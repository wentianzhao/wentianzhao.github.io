gateway网关总结

发表于 2019-11-15 更新于 2019-11-18 分类于 项目总结 ， spring cloud 阅读次数：

项目描述：

功能设计及开发流程：

  • 添加表：

    应用信息表：（参考system表，添加系统给生成的key，应用管理企业名称）

    系统接口信息表：（系统编码，接口名/xxx，系统地址）

    应用系统接口对应表（应用名，接口id，接口名，系统名）

  • 流程：

    1. 会给不同的企业分配一个key，企业调用接口时把key、系统编码，时间戳，密钥放header传过来
    2. 由网关（spring cloud gateway）接收，拦截后从权限管理获取token，匹配加密对不对，之后再获取该应用能调用的所有接口，网关获取后保存到缓存中，方便下次调用，如果两步鉴定正确，再把接口分配到所属系统

  • 开发流程：

    1. 建表
    2. 权限管理中添加应用管理，（查询条件：应用编码，应用名称，）功能：生成key，添加系统接口权限（要能够根据系统查询），
    3. 开发网关系统进行鉴权
    4. 系统间调用，需要加一个共用sdk，方便授权其他系统调用

  • 好处：

    1. 接口级控制，能够对不同系统调用本企业系统接口的调用进行控制，方便进行授权管理；

    2. 统一第三方系统调用入口，由gateway获取后分发；

    3. 能够记录接口调用日志，记录不同企业系统间的调用，

职责描述：

  • 开发系统权限管理功能，给系统赋接口权限；
  • 开发spring cloud gateway，通过使用feign调用权限管理接口，获取应用权限，保存到redis中，降低接口调用时间；
  • 开发SDK包，集成接口加密规则，方便第三方系统调用；
  • 调整业务系统通过gateway调用权限管理系统接口，已应用到线上生产环境；

技术描述：

项目使用spring cloud gateway框架开发，集成spring cloud feign进行系统接口调用，前期有集成Eurake服务，进行第三方接口调用时的系统接口权限验证，通过jar包方式部署，目前已有三个业务系统调用，整体运行正常。

1 概述

检查管理系统V4.0（以下简称证后监管系统）提供获取化妆品经营企业检查信息、药品零售企业检查信息、医疗机构信息等Open API接口，提供相关企业的检查信息供第三方系统使用。

1.1 Open API 说明

Open API 提供外部应用系统通过主动调用的方式进行访问，当你的应用调用开放平台接口时，需使用 HTTPS 协议、JSON 数据格式、UTF8 编码。

系统管理员可通过证后监管系统对需要对接的外部应用系统进行授权和发放相关密钥信息，通过密钥可以生成每次调用接口的签名sign,密钥不会在请求调用中传输，请妥善保存。

sign是用来验证相关系统访问证后监管系统合法性的凭证，当外部系统调用证后监管系统接口时，证后监管系统会根据提交的AppId校验访问的合法性以及所对应的权限并返回相应的结果。

1.2 访问频次控制说明

Open API 的访问频次不能超过 1000次/每分钟。

2 请求说明

证后监管系统提供正式环境地址，需申请AppId和AppSecret，调用开放平台的任何接口都需要携带sign和AppId。请求响应格式目前仅支持json。所有请求需签名，签名算法目前仅支持MD5，具体签名规则在2.4中描述。

2.2 公共请求头（Header）

参数 类型 是否必填 描述
appId String 是 证后监管系统分配给开发者的应用ID
timestamp String 是 时间戳，格式为yyyy-MM-dd HH:mm:ss，时区为GMT+8，例如：2019-07-01 12:00:00。允许客户端请求最大时间误差为10分钟。
sign String 是 API输入参数签名结果，签名算法参照下面的介绍

2.3 公共响应参数

参数 类型 是否必填 描述
returnCode String 是 “0”表示正常返回，“1”查看returnMessage信息，“-1”服务器异常
returnMessage String 是 业务提示信息
returnDate String 否 具体接口返回数据

2.4 签名算法

为了防止API调用过程中被黑客恶意篡改，调用任何一个API都需要携带签名，证后监管会根据请求参数，对签名进行验证，签名不合法的请求将会被拒绝。目前仅支持MD5算法，签名大体过程如下：

· 对所有API请求参数（包括公共参数和业务参数，但除去sign参数），根据参数名称的ASCII码表的顺序排序。如：foo:1, bar:2, foo_bar:3, foobar:4排序后的顺序是bar:2, foo:1, foo_bar:3, foobar:4。

· 将排序好的参数名和参数值拼装在一起，根据上面的示例得到的结果为：bar2foo1foo_bar3foobar4。

· 把拼装好的字符串采用utf-8编码，使用签名算法对编码后的字节流进行摘要。如果使用MD5算法，则需要在拼装的字符串前后加上app的secret后，再进行摘要，如：md5(secret+bar2foo1foo_bar3foobar4+secret)。

· 将摘要得到的字节流结果使用十六进制表示，如：hex(“helloworld”.getBytes(“utf-8”)) = “68656C6C6F776F726C64”
