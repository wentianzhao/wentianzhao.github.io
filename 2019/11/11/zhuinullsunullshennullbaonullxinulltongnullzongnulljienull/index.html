追溯申报系统总结

发表于 2019-11-11 更新于 2019-11-12 阅读次数：

系统设计分析

系统采用分布式部署的方式，用微服务的形式设计。

系统整体为追溯申报系统，子系统：器械召回系统，网络备案系统，自查报告系统，三方平台自查上报系统；服务系统分为权限管理，产品管理，企业管理，短信发送平台，参数管理，gateway网关。

系统用到的技术：spring boot，spring cloud （gateway，feign），restful，spring date jpa，lombok，redis，joda-time，shiro，schedule，logback，poi。

业务介绍：

监管端：提供企业管理、企业购买销售产品管理（产品，上下游企业），产品管理，企业公告留言。

  1. 提供全市企业相关信息查询，修改监管部门，发动短信，导出功能；
  2. 提供企业购销信息查询，产品经营情况查询，

企业端：

  1. 企业基本信息管理，人员管理，

  2. 新增产品申报，产品购销信息管理

登录过程：

  1. 填写用户名密码，前端通过DES加密，船只后端，后端通过自定义密钥解密，获取用户名密码后，交给shiro获取用户权限角色等信息；
  2. 在用户请求链接时，会首先进行shiro拦截并鉴权，获取当前session中是否有用户；
  3. 获取后通过shiro进行登录，在跳转至链接
  • cookie是保存在客户端，session保存在服务器
# 总结
