追溯申报系统总结

发表于 2019-11-11 更新于 2019-11-16 分类于 项目总结 阅读次数：

项目描述：

项目为器械经营企业产品追溯申报系统，分为企业端和监管端。

企业端主要负责本企业购销产品和申报新增产品，管理企业人员信息，上报企业自查信息，网络销售备案信息，对已销售产品进行召回，查看监管端公告和对监管端留言等功能；

监管端分为企业信息管理，企业上报信息管理，上报产品查询，发送短信，审核企业自查报告，主动召回产品报告，网络备案信息，发布公告和查看留言等。

职责描述：

  • 开发企业和监管用户权限管理功能，修复用户权限漏洞；
  • 优化产品购销关系导入导出功能，代码优化，sql优化；
  • 优化注册、登录功能，添加滑动框校验登录；
  • 开发企业检查信息上报功能，监管端信息管理功能；
  • 独立负责后期日常系统维护，新增需求开发；
  • 修复系统安全检索出的漏洞；

技术分析：

整体为追溯申报系统，子系统分为：器械召回系统，网络备案系统，自查报告系统，三方平台自查上报系统；服务系统分为权限管理，产品管理，企业管理，短信发送平台，参数管理，gateway网关。

系统用到的技术：spring boot，restTemplate API，spring date jpa，lombok，redis，joda-time，shiro，scheduled，logback，前端模版thymeleaf，Excel文件导入导出，sso单点登录，nginx。

系统采用spring boot框架开发，分布式部署的方式，将底层支撑拆分为微服务形式，不同系统之间调用通过restful api形式，底层通过spring data jpa和spring jdbc调用数据，实体类通过lombok优化基础方法。登录时通过调用权限管理获取登录用户权限，注册时用redis保存手机验证码。通过ScheduledTasks执行定时任务，多线程解析用户导入Excel文件。上报企业过多时通过nginx进行负载均衡，线上问题通过arthas排错，tomcat方式部署。

登录过程：

  1. 填写用户名密码，前端通过DES加密，船只后端，后端通过自定义密钥解密，获取用户名密码后，交给shiro获取用户权限角色等信息；
  2. 在用户请求链接时，会首先进行shiro拦截并鉴权，获取当前session中是否有用户；
  3. 获取后通过shiro进行登录，在跳转至链接
  • cookie是保存在客户端，session保存在服务器
